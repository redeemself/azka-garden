## 🌿 Tentang Azka Garden

Azka Garden adalah aplikasi **e-commerce toko bunga** yang dibangun menggunakan **Laravel PHP Framework** dengan desain modern dan user-friendly. Aplikasi ini dirancang khusus untuk usaha hortikultura keluarga Pak Hendrik yang berlokasi di Depok, Jawa Barat, menggabungkan kios tanaman fisik dengan kanal edukasi daring di WhatsApp, Tokopedia, Google Maps, dan Google.

### **📍 Informasi Bisnis**

**Lokasi Pusat**: HRQH+3VP, Jalan Raya KSU, Kelurahan Tirtajaya, Kecamatan Sukmajaya, Depok  
**Kontak**: 0896-3508-6182  
**Operasional**: 24 jam setiap hari  
**Fasilitas**: Kios bibit, pot, pupuk, dan perlengkapan taman dengan lapak fisik berbentuk los terbuka dengan atap paranet khas nursery tepi jalan Depok–Parung.

Usaha ini terkenal dan namanya dipakai oleh usaha serupa di berbagai kota lain (Yogyakarta, Pekanbaru, Batam, Tangerang), sehingga Azka Garden kini mencakup penjualan tanaman, jasa lanskap, dan produksi konten hortikultura di Indonesia.

## **🔧 Detail Implementasi**

### **1. Konfigurasi Authentication Multi-Guard**

Repository ini menggunakan sistem authentication multi-guard yang mendukung:

-   **User** (untuk pelanggan biasa)
-   **Admin** (untuk admin toko)
-   **Developer** (untuk developer/superadmin)

```php

    'web' => ['driver' => 'session', 'provider' => 'users'],
    'admin' => ['driver' => 'session', 'provider' => 'admins'],
    'developer' => ['driver' => 'session', 'provider' => 'developers'],
```

## **🛠️ Troubleshooting & Common Errors**

### **1. Route Error: user.address.store not defined**

If you encounter this error:

```text
Symfony\Component\Routing\Exception\RouteNotFoundException
Route [user.address.store] not defined.
```

Add the following code to your `routes/web.php` file:

```php
use App\Http\Controllers\User\AddressController;

Route::middleware(['auth'])->group(function () {
    // Add resource route for addresses
    Route::resource('user/address', AddressController::class);
});
```

Create the controller if it doesn't exist:

```bash
php artisan make:controller User/AddressController --resource
```

And implement the controller with store method:

```php
<?php

namespace App\Http\Controllers\User;

use App\Http\Controllers\Controller;
use Illuminate\Http\Request;

class AddressController extends Controller
{
    public function store(Request $request)
    {
        // Validate the incoming request
        $validated = $request->validate([
            'address' => 'required|string|max:255',
            'city' => 'required|string|max:100',
            'state' => 'required|string|max:100',
            'postal_code' => 'required|string|max:20',
        ]);

        // Store the address
        auth()->user()->addresses()->create($validated);

        // Redirect back with success message
        return redirect()->back()->with('success', 'Address added successfully!');
    }

    // Implement other resource methods as needed
}
```

### **2. Lodash Module Resolution Error**

If you encounter this error:

```text
Uncaught TypeError: Failed to resolve module specifier "lodash".
Relative references must start with either "/", "./", or "../".
```

Use one of these solutions:

#### **Option 1: Quick Fix with Import Map (Recommended)**

Add an import map to your HTML file:

```html
<script type="importmap">
    {
        "imports": {
            "lodash": "https://cdn.skypack.dev/lodash"
        }
    }
</script>
```

Or use this command:

```bash
sed -i '/<head>/a \    <script type="importmap">\n    {\n      "imports": {\n        "lodash": "https://cdn.skypack.dev/lodash"\n      }\n    }\n    </script>' index.html
```

#### **Option 2: Using a Bundler (More Comprehensive)**

```bash
# Navigate to your project directory
cd azka-garden

# Install dependencies including lodash
npm install lodash

# Install Vite bundler
npm install --save-dev vite

# Create or update package.json scripts
echo '{
  "scripts": {
    "dev": "vite",
    "build": "vite build"
  }
}' > package.json

# Create a vite.config.js file
echo 'export default {
  root: ".",
  build: {
    outDir: "dist"
  }
}' > vite.config.js

# Run the dev server
npm run dev
```

#### **Option 3: Update Import Statements Directly**

```bash
# Find all JS files with lodash imports and update them
find . -name "*.js" -type f -exec sed -i 's/import _ from "lodash";/import _ from "https:\/\/cdn.skypack.dev\/lodash";/g' {} \;
```

## **📋 Panduan Push Repository ke GitHub**

Berikut ringkasan lengkap dan urut langkah benar menambahkan folder lokal azka-garden beserta isinya ke GitHub repository redeemself/azka-garden:

### **1. Pastikan Repository Sudah Ada di GitHub**

Arahkan ke folder induk tempat azka-garden berada:

```bash
cd path\to\azka-garden
```

### **3. Inisialisasi Git (jika belum pernah git init)**

```bash
git init
```

### **4. Tambahkan Remote Origin (hubungkan ke repo GitHub)**

```bash
git remote add origin https://github.com/redeemself/azka-garden.git
```

Jika sudah ada remote dan perlu update URL:

```bash
git remote set-url origin https://github.com/redeemself/azka-garden.git
```

### **5. Tambahkan Semua File dan Folder ke Staging Area**

```bash
git add .
```

### **6. Commit Perubahan dengan Pesan Jelas**

```bash
git commit -m "Initial commit: add azka-garden project"
```

### **7. Push ke Repository GitHub**

#### **Jika repo GitHub masih kosong (fresh), cukup:**

```bash
git branch -M main
git push -u origin main
```

#### **Jika remote sudah berisi commit dan muncul error non-fast-forward:**

```bash
git pull origin main --rebase
# Selesaikan konflik jika ada, lalu:
git push -u origin main
```

#### **Jika yakin ingin menimpa isi remote dengan isi lokal (hati-hati!):**

